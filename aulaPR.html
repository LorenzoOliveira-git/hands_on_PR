Sobre o Projeto

Este projeto tem como objetivo ensinar o fluxo de trabalho do GitHub, incluindo a criação de branches, commits, Pull Requests e revisões de código.

Funcionalidades

  • Visualização de páginas HTML simples.
  • Interação com formulários.
  • Design simples e responsivo.

Pull Requests

Pull Request é uma maneira de trabalhar com grupos de desenvolvimento. Sua principal diferença é a revisão de alterações, onde o admin da aplicação irá aceitar ou rejeitar suas features novas. Aqui você aprenderá o conceito e entenderá na prática como esse estilo de organização funciona!

Merge

Merge é uma maneira de juntar alterações feitas entre integrantes do grupo. Sua principal diferença é a integração entre diferentes branch do projeto, mantendo uma organização modular. Vamos aprender na teoria e na prática como esse estilo de organização funciona!
