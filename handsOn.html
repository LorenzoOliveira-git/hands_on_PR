Grupo

Chame seus amigos para praticar essa atividade, é necessário no mínimo três pessoas para realizar essa atividade.

Funções

Líder: deve haver apenas um por grupo, será responsável por coordenar o grupo, revisar as PRs de todos e garantir que o processo esteja sendo seguido.

Desenvolvedores: são os outros integrantes, que são responsáveis por implementar uma funcionalidade mais complexa ou por definir uma parte importante do projeto (ex: criar o layout de uma página ou adicionar uma nova feature) e testar a aplicação localmente.

Repositório

Deve cirar um repoitório (remoto e local) e inserir o arquivo html que pode ser acessado pelo botãoabaixo e compartilhar com os colegas do grupo

Clique aqui para acessar o arquivo html

Branchs

Cada desenvolvedor deve criar uma Branch para criar alterações no projeto. A Branch deve ter um verbo e um nome relacionado à alteração, por exemplo: adicionar-formulário

Commits

Cada membro trabalha na sua parte do código e ao concluir as alterações, eles fazem commit regularmente, com mensagens de commit claras, como:

  • "Adiciona formulário de contato"
  • "Ajusta layout da página de contato"

Pull Requests

Depois de realizar os commits, os integrantes devem enviar suas alterações (push) para o GitHub. Isso torna as alterações acessíveis aos outros membros do grupo.

Início revisão

O Líder do grupo irá acessar os Pull Requests criados por cada membro. Ele deve revisar o código de todos os colegas e deixar comentários construtivos e sugestões de melhoria.

Término revisão

Quando as alterações forem aprovadas (após discussões e ajustes), o Líder de Projeto pode realizar o merge da branch de cada membro com a branch principal
